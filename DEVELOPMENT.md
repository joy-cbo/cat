## 1. 项目概述

一个帮助猫主人全面管理猫咪健康、日常护理和生活记录的 Web 应用。部署在 Cloudflare 全栈平台上。

### 目标用户
- 个人猫主人（1-5只猫）
- 多猫家庭
- 猫舍管理员（可选扩展）

### 核心价值
- 集中管理所有猫咪的健康档案
- 不再忘记驱虫、疫苗等重要时间节点
- 记录猫咪成长轨迹，数据可视化

## 2. 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **前端框架** | Nuxt 3 | Vue 3 官方元框架，支持 SSR/SSG |
| **UI 组件** | Naive UI | Vue 3 组件库，中文友好 |
| **样式** | Tailwind CSS | 原子化 CSS |
| **后端** | Nitro (Nuxt 内置) | 原生支持 Cloudflare Workers/Pages |
| **数据库** | Cloudflare D1 | SQLite on Edge，免费额度大 |
| **ORM** | Drizzle ORM | 类型安全，D1 原生支持 |
| **文件存储** | Cloudflare R2 | 图片/附件，零出口费 |
| **缓存** | Cloudflare KV | 会话/配置缓存 |
| **认证** | 自写 JWT | 轻量，httpOnly cookie 存储 |
| **图表** | ECharts / Chart.js | 数据可视化 |
| **部署** | Cloudflare Pages | 前后端一体部署 |

## 3. 功能模块详述

### 2.1 猫咪档案管理

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 名字 | text | ✅ | 猫咪昵称 |
| 品种 | select | ❌ | 英短、美短、布偶、暹罗、田园猫等 |
| 性别 | radio | ✅ | 公/母 |
| 出生日期 | date | ❌ | 用于计算年龄 |
| 颜色/花纹 | text | ❌ | 橘猫、三花、重点色等 |
| 体重(kg) | number | ❌ | 最新体重 |
| 头像 | image | ❌ | 上传至 R2 |
| 入家日期 | date | ❌ | 什么时候来到你身边 |
| 状态 | select | ✅ | 在家/送养/离世 |
| 芯片号 | text | ❌ | 猫咪身份标识 |
| 备注 | textarea | ❌ | 性格、喜好、过敏原等 |

**功能点：**
- 猫咪列表页（卡片式展示，支持搜索/筛选）
- 猫咪详情页（档案 + 健康时间线 + 相册）
- 支持多张照片（R2 存储）
- 批量导入（CSV）

### 2.2 健康记录管理

#### 2.2.1 记录类型

| 类型 | 频率参考 | 必记字段 |
|------|----------|----------|
| **疫苗** | 每年/首次3针 | 疫苗名称、批次、接种机构、下次接种日期 |
| **驱虫** | 内驱3月/外驱1月 | 药品名称、用药剂量、下次驱虫日期 |
| **绝育** | 一次性 | 手术日期、医院、术后恢复情况 |
| **就医** | 按需 | 就诊原因、诊断结果、处方、费用、复诊日期 |
| **体检** | 每年1-2次 | 体检项目、结果异常项、建议 |
| **体重记录** | 按需 | 体重值、日期（用于趋势图） |
| **刷牙/洗耳** | 按需 | 护理项目、使用的物品 |

#### 2.2.2 功能点
- 每只猫独立的健康时间线
- 支持上传化验单/处方照片
- 费用统计（按猫、按时间范围）
- 健康趋势图表（体重曲线、疫苗覆盖）

├── / (首页/仪表盘)

## 5. 非功能需求

### 5.1 一键部署

提供小白式一键部署能力，用户无需懂技术即可完成部署：

| 部署方式 | 说明 |
|----------|------|
| **GitHub + Cloudflare** | 推送代码到 GitHub，Cloudflare 自动部署 |
| **Docker 本地部署** | 提供 `docker-compose.yml`，一键启动 |

**GitHub → Cloudflare 部署流程：**
1. 用户 Fork 仓库到自己的 GitHub
2. 在 Cloudflare Pages 关联 GitHub 仓库
3. 设置构建命令和输出目录
4. 配置 D1、R2 绑定
5. Push 代码 → Cloudflare 自动构建部署
6. 用户访问 → 注册首个账号 → 自动成为管理员

**Cloudflare Pages 配置：**
```
构建命令: npm run build
输出目录: .output/public
Node.js 版本: 18+
```

**GitHub Actions 自动化：**
- Push 到 main → 自动部署到 Cloudflare
- PR → 自动运行 lint + test
- Release → 自动创建生产部署

**需要准备：**
- `.github/workflows/deploy.yml` CI/CD 配置
- `wrangler.toml` 配置模板
- README 部署说明
- `docker-compose.yml` 本地部署方案（备选）

### 5.2 管理员机制

| 规则 | 说明 |
|------|------|
| **首个注册用户** | 自动成为管理员（admin） |
| **后续注册** | 普通用户（member） |
| **管理员关闭** | 管理员可在设置中关闭公开注册 |
| **邀请码机制** | 关闭注册后，通过邀请码加入 |

**实现逻辑：**
```
用户注册 → 检查 users 表是否为空
  ├── 空 → 角色设为 admin，开放注册
  └── 非空 → 检查注册开关
        ├── 开放 → 角色设为 member
        └── 关闭 → 需要邀请码
```

**数据库字段：**
```sql
-- 系统设置表
CREATE TABLE settings (
  key TEXT PRIMARY KEY,
  value TEXT NOT NULL,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 默认设置
INSERT INTO settings (key, value) VALUES ('registration_open', 'true');
```
