Cloudflare 一键部署指南

跟着步骤操作，5分钟完成部署！

📋 部署前准备

📧
GitHub 账号
没有就去注册一个
☁️
Cloudflare 账号
免费注册即可
💻
电脑终端
Mac/Windows 都行

1 Fork 项目到你的 GitHub

1. 打开项目页面：

👉 https://github.com/joy-cbo/cat

2. 点击右上角的 "Fork" 按钮

就是那个分叉形状的图标 ↗️

3. 点击绿色的 "Create fork" 按钮

完成后你的 GitHub 上就有了 你的用户名/cat 仓库

2 注册 Cloudflare 账号

打开注册页面：

👉 https://dash.cloudflare.com/sign-up

填写邮箱和密码，点击 "Create account"，完成邮箱验证即可

3 安装 Wrangler CLI 工具

Mac 用户：
npm install -g wrangler
Windows 用户（在 PowerShell 中运行）：
npm install -g wrangler
💡
如果没有安装 Node.js，先去 nodejs.org 下载安装

4 登录 Cloudflare

wrangler login

浏览器会自动打开，点击 "Allow" 授权即可

5 创建 Cloudflare 资源

在终端依次运行以下命令：

# 1. 创建数据库
wrangler d1 create cat-manager-db
📝
运行后会显示 database_id = "xxxx"，复制这个 ID 保存好！
# 2. 创建存储桶
wrangler r2 bucket create cat-manager-r2
# 3. 创建 KV 命名空间
wrangler kv namespace create KV
📝
运行后会显示 id = "xxxx"，复制这个 KV ID 保存好！

6 获取 Cloudflare 账号信息

获取 Account ID：

  1. 打开 Cloudflare 控制台
  2. 点击右侧边栏的 "Workers & Pages"
  3. 右侧会显示 "Account ID"
  4. 复制这个 ID 保存好！

获取 API Token：

  1. 打开 API Tokens 页面
  2. 点击 "Create Token"
  3. 选择 "Cloudflare Pages - Edit" 模板
  4. 点击 "Continue to summary"
  5. 点击 "Create Token"
  6. 复制这个 Token 保存好！（只显示一次）

7 配置 GitHub Secrets

  1. 打开你的仓库 https://github.com/你的用户名/cat
  2. 点击 "Settings" 标签
  3. 左侧点击 "Secrets and variables" → "Actions"
  4. 点击 "New repository secret"
Name Value
CLOUDFLARE_API_TOKEN 第 6 步复制的 API Token
CLOUDFLARE_ACCOUNT_ID 第 6 步复制的 Account ID

8 配置数据库 ID

  1. 克隆你的仓库到本地：
    git clone https://github.com/你的用户名/cat.git
    cd cat
  2. 用文本编辑器打开 wrangler.toml 文件
  3. 修改以下内容（替换成你自己的 ID）：
[[d1_databases]]
binding = "DB"
database_name = "cat-manager-db"
database_id = "粘贴你的 database_id"  # ← 替换

[[r2_buckets]]
binding = "R2"
bucket_name = "cat-manager-r2"

[[kv_namespaces]]
binding = "KV"
id = "粘贴你的 KV ID"  # ← 替换

[vars]
JWT_SECRET = "改成一个复杂的密码"  # ← 改成你自己的

保存后提交代码：

git add wrangler.toml
git commit -m "配置 Cloudflare 资源 ID"
git push

9 等待自动部署

✅
推送代码后，GitHub Actions 会自动完成部署！

查看部署状态：

  1. 打开你的 GitHub 仓库
  2. 点击 "Actions" 标签
  3. 可以看到正在进行的部署

获取访问地址：

  1. 打开 Cloudflare 控制台
  2. 点击 "Workers & Pages"
  3. 找到 "cat-manager" 项目
  4. 点击进入，可以看到你的网站地址！

10 🎉 部署完成！

首次访问设置：

  1. 点击 Cloudflare 给你的网站地址
  2. 第一个注册的用户会自动成为管理员！
  3. 填写邮箱和密码，点击注册

❓ 常见问题

部署失败怎么办？
  1. 检查 GitHub Secrets 是否正确
  2. 检查 wrangler.toml 中的 ID 是否正确
  3. 在 Actions 页面查看错误日志
如何更新代码？
git pull && git push 推送后会自动重新部署！
需要付费吗？
Cloudflare 免费套餐完全够用：D1 5GB、R2 10GB、Pages 无限部署
数据会丢失吗？
不会！Cloudflare D1 会自动备份你的数据。
