Fork 项目（复制一份到你自己的 GitHub）

打开下面的链接，然后点击右上角的 "Fork" 按钮：

📦 打开项目页面 →

操作说明：

① 点击页面右上角的 Fork 按钮（分叉图标 ↗️）
② 点击绿色的 创建 Fork 按钮
2

注册 Cloudflare 账号

如果已有 Cloudflare 账号，直接跳到第 3 步：

☁️ 打开 Cloudflare 注册页 →
3

创建 Cloudflare Pages 项目

🔗 打开 Cloudflare 控制台 →

操作说明：

① 左侧点击 Workers 和 Pages
② 点击 "创建" 按钮
③ 选择 "Pages" 标签
④ 点击 "连接到 Git"
⑤ 选择 GitHub，授权 Cloudflare 访问你的 GitHub
⑥ 搜索并选择你的 cat 仓库
⑦ 设置如下，然后点击 "保存并部署"：
生产分支：
main
构建命令：
npm run build
构建输出目录：
.output/public
4

创建数据库（D1）

操作说明：

① 左侧点击 存储和数据库
② 点击 D1 SQL 数据库
③ 点击 "创建数据库"
④ 数据库名称填：cat-manager-db
⑤ 位置选离你最近的（如 东亚 或 亚太）
⑥ 点击 "创建数据库"
5

绑定数据库到 Pages

操作说明：

① 左侧点击 Workers 和 Pages
② 点击你的 cat-manager 项目
③ 点击 "设置" 标签
④ 点击左侧 "函数"
⑤ 滚动到底部找到 "D1 数据库绑定"
⑥ 点击 "添加绑定"
⑦ 变量名称填：DB，数据库选 cat-manager-db
⑧ 点击 "保存"
6

创建文件存储（R2）

操作说明：

① 左侧点击 R2 对象存储
② 点击 "创建存储桶"
③ 存储桶名称填：cat-manager-r2
④ 位置选离你最近的（如 东亚 或 亚太）
⑤ 点击 "创建存储桶"
7

绑定 R2 到 Pages

操作说明：

① 回到你的 cat-manager 项目
② 点击 "设置" 标签
③ 点击左侧 "函数"
④ 滚动到底部找到 "R2 存储桶绑定"
⑤ 点击 "添加绑定"
⑥ 变量名称填：R2，存储桶选 cat-manager-r2
⑦ 点击 "保存"
8

创建缓存（KV）

操作说明：

① 左侧点击 存储和数据库
② 点击 "KV" 标签
③ 点击 "创建命名空间"
④ 命名空间名称填：KV
⑤ 点击 "添加"
9

绑定 KV 到 Pages

操作说明：

① 回到你的 cat-manager 项目
② 点击 "设置" 标签
③ 点击左侧 "函数"
④ 滚动到底部找到 "KV 命名空间绑定"
⑤ 点击 "添加绑定"
⑥ 变量名称填：KV，命名空间选 KV
⑦ 点击 "保存"
10

设置环境变量

操作说明：

① 回到你的 cat-manager 项目
② 点击 "设置" 标签
③ 点击左侧 "环境变量"
④ 点击 "添加变量"
⑤ 变量名称填：JWT_SECRET
⑥ 值填一个随机字符串（如 cat-abc123xyz）
⑦ 点击 "保存"
11

重新部署

⚠️ 绑定资源后必须重新部署才能生效！

操作说明：

① 点击 "部署" 标签
② 找到最新的一次部署
③ 点击右侧 ... 菜单
④ 点击 "重新部署"
⑤ 等待部署完成...
🎉

❓ 常见问题

部署失败怎么办？

1. 检查绑定的名称是否正确（必须是 DB、R2、KV）

2. 在部署标签页查看部署日志

3. 绑定完成后记得重新部署

需要花钱吗？
完全免费！Cloudflare 免费套餐：D1 数据库 5GB、R2 存储 10GB、Pages 无限部署。
数据会丢失吗？
不会！Cloudflare D1 自动备份数据，非常安全。
怎么更新网站？
在 GitHub 修改代码并推送，Cloudflare 会自动重新部署。
手机能用吗？
可以！部署后手机浏览器直接访问网址即可使用。
