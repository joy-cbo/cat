# 🐱 猫咪管理系统 - 一键部署指南

> ✅ 免费 | ✅ 无需安装软件 | ✅ 全程网页操作

---

## 📋 部署前准备

你需要准备：
1. 一个 [GitHub](https://github.com) 账号
2. 一个 [Cloudflare](https://dash.cloudflare.com/sign-up) 账号（免费注册）

---

## 第 1 步：Fork 项目

> 把项目复制一份到你自己的 GitHub

1. 打开 👉 [https://github.com/joy-cbo/cat](https://github.com/joy-cbo/cat)
2. 点击右上角的 **Fork** 按钮（分叉图标 ↗️）
3. 点击绿色的 **创建 Fork** 按钮

完成后你的 GitHub 上就有了 `你的用户名/cat` 仓库

---

## 第 2 步：注册 Cloudflare

> 如果已有账号，直接跳到第 3 步

1. 打开 👉 [https://dash.cloudflare.com/sign-up](https://dash.cloudflare.com/sign-up)
2. 输入邮箱和密码
3. 点击 **创建账户**
4. 去邮箱点击验证链接

---

## 第 3 步：创建 Cloudflare Pages 项目

1. 打开 👉 [https://dash.cloudflare.com](https://dash.cloudflare.com)
2. 左侧点击 **Workers 和 Pages**
3. 点击 **创建** 按钮
4. 选择 **Pages** 标签
5. 点击 **连接到 Git**
6. 选择 **GitHub**，授权 Cloudflare 访问你的 GitHub
7. 搜索并选择你的 **cat** 仓库
8. 设置如下，然后点击 **保存并部署**：

| 设置项 | 值 |
|--------|-----|
| 生产分支 | `main` |
| 构建命令 | `npm run build` |
| 构建输出目录 | `.output/public` |

等待部署完成...

---

## 第 4 步：创建数据库（D1）

1. 左侧点击 **存储和数据库**
2. 点击 **D1 SQL 数据库**
3. 点击 **创建数据库**
4. 数据库名称填：`cat-manager-db`
5. 位置选离你最近的（如 **东亚** 或 **亚太**）
6. 点击 **创建数据库**

---

## 第 5 步：绑定数据库到 Pages

1. 左侧点击 **Workers 和 Pages**
2. 点击你的 **cat-manager** 项目
3. 点击 **设置** 标签
4. 点击左侧 **函数**
5. 滚动到底部找到 **D1 数据库绑定**
6. 点击 **添加绑定**
7. 设置如下：

| 设置项 | 值 |
|--------|-----|
| 变量名称 | `DB` |
| D1 数据库 | 选择刚才创建的 `cat-manager-db` |

8. 点击 **保存**

---

## 第 6 步：创建文件存储（R2）

1. 左侧点击 **R2 对象存储**
2. 点击 **创建存储桶**
3. 存储桶名称填：`cat-manager-r2`
4. 位置选离你最近的（如 **东亚** 或 **亚太**）
5. 点击 **创建存储桶**

---

## 第 7 步：绑定 R2 到 Pages

1. 回到你的 **cat-manager** 项目
2. 点击 **设置** 标签
3. 点击左侧 **函数**
4. 滚动到底部找到 **R2 存储桶绑定**
5. 点击 **添加绑定**
6. 设置如下：

| 设置项 | 值 |
|--------|-----|
| 变量名称 | `R2` |
| R2 存储桶 | 选择刚才创建的 `cat-manager-r2` |

7. 点击 **保存**

---

## 第 8 步：创建缓存（KV）

1. 左侧点击 **Workers 和 Pages**
2. 点击顶部 **KV** 标签
3. 点击 **创建命名空间**
4. 命名空间名称填：`KV`
5. 点击 **添加**

---

## 第 9 步：绑定 KV 到 Pages

1. 回到你的 **cat-manager** 项目
2. 点击 **设置** 标签
3. 点击左侧 **函数**
4. 滚动到底部找到 **KV 命名空间绑定**
5. 点击 **添加绑定**
6. 设置如下：

| 设置项 | 值 |
|--------|-----|
| 变量名称 | `KV` |
| KV 命名空间 | 选择刚才创建的 `KV` |

7. 点击 **保存**

---

## 第 10 步：设置环境变量

1. 回到你的 **cat-manager** 项目
2. 点击 **设置** 标签
3. 点击左侧 **环境变量**
4. 点击 **添加变量**
5. 添加以下变量：

| 变量名称 | 值 |
|----------|-----|
| `JWT_SECRET` | 随机字符串（如 `cat-abc123xyz`） |

6. 点击 **保存**

---

## 第 11 步：重新部署

绑定完成后需要重新部署：

1. 点击 **部署** 标签
2. 找到最新的一次部署
3. 点击右侧 **...** 菜单
4. 点击 **重新部署**
5. 等待部署完成

---

## 🎉 部署成功！

你的猫咪管理系统已经上线了！

### 下一步：

1. 点击顶部的 **访问站点** 按钮打开你的网站
2. 点击 **注册**
3. 第一个注册的用户自动成为 **管理员**
4. 开始添加你的猫咪吧！🐱

> 💡 以后更新代码，只需在 GitHub 修改并推送，网站会自动更新！

## ❓ 常见问题

### 部署失败怎么办？

1. 检查绑定的名称是否正确（必须是 `DB`、`R2`、`KV`）
2. 在部署标签页查看部署日志
3. 绑定完成后记得重新部署

### 需要花钱吗？

完全免费！Cloudflare 免费套餐：D1 数据库 5GB、R2 存储 10GB、Pages 无限部署。

### 数据会丢失吗？

不会！Cloudflare D1 自动备份数据，非常安全。

### 怎么更新网站？

在 GitHub 修改代码并推送，Cloudflare 会自动重新部署。

### 手机能用吗？

可以！部署后手机浏览器直接访问网址即可使用。
